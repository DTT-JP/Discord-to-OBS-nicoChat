# トラブルシューティング

運用中によくある問題を、確認しやすい順でまとめています。  
最終的な挙動はソースコードを正としてください。

> 公開BOT利用時は、まず Discord 側の権限設定（`/config`・`/setup`）を確認してください。  
> セルフホスト時のみ、`.env` や HTTPS/CORS 設定の確認が必要です。

## 認証コードが表示されない

- OBS のブラウザソースを右クリックして **更新** を実行する
- Node.js 側ログに接続成功ログ（例: `[manager] 接続確立`）が出ているか確認する
- `HOST` / `PORT` または `PUBLIC_URL` が OBS から到達可能か確認する

## `/start` が権限エラーになる

次のいずれかを満たす必要があります。

- サーバーオーナー、または管理者（Administrator）
- `/setup allow_start_role` または `/setup allow_start_user` で許可済み

補足: `/start` 許可の設定は `/config` ではなく `/setup` 側です。

## `/start` がコマンド一覧に出ない

- Bot が `commands/start.js` を含むバージョンで起動しているか確認する
- `npm run deploy-commands` を再実行する
- グローバル反映は最大約1時間かかる場合がある

## `/auth` が実行できない

- `/auth` は DM ではなく、`/start` と同じサーバーのチャンネルで実行する
- 認証失敗を5回続けると5分間ロックされるため、時間をおいて再試行する
- それでも通らない場合は `/start` からやり直す

## 一覧ボタン（◀ ▶）でエラーになる

- Bot を最新版に更新する
- `npm run deploy-commands` を再実行する
- 旧メッセージのボタンは無効化される場合がある

## Bot がチャンネルを読めない

Discord のチャンネル権限で、Bot ロールに以下を付与してください。

- チャンネルを見る
- メッセージ履歴を読む

## コメントが流れない・表示されない

- `/auth` が完了しているか確認する（認証画面が消えている状態）
- ブラウザ開発者ツール（F12）のコンソールエラーを確認する
- Bot 側ログにエラーが出ていないか確認する

## `deploy-commands` が失敗する

`.env` の `DISCORD_TOKEN` と `CLIENT_ID` を再確認してください。

## HTTPS運用なのに `http://` URL が出る

`PUBLIC_URL` を設定して Bot を再起動してください。

```env
PUBLIC_URL=https://overlay.example.com
```

## スタンプがDiscordチャット欄に出ない

仕様です。スタンプ画像は OBS オーバーレイ表示のみ対応です。

## 再起動・PM2 reload 時のセッション

グレースフル終了時はセッション行を保持し、`socket_id` をクリアします。  
条件によっては `sessionStorage` の再開トークンで再接続される場合があります。
