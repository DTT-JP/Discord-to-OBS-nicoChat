# セキュリティ

このドキュメントは運用時に必要なセキュリティ要点をまとめたものです。  
公開BOT利用時はサービス提供側の運用ポリシーが優先され、セルフホスト時は本ドキュメントの項目を運用者責任で管理します。  
最終的な仕様は常にソースコード（`index.js` / `events/interactionCreate.js` / `commands/*.js`）を正とします。

## 認証フロー

### ワンタイムトークン

`/start` 実行時に UUID v4 のワンタイムトークンを発行します。  
トークンには有効期限（既定: 10 分）があり、期限切れは定期クリーンアップで削除されます。

### 6桁コード照合

ブラウザ接続後に発行される 6 桁コードは、`/start` 実行者の Discord ID と照合されます。  
URL を知っていても、別アカウントからは認証できません。

`/auth` は **DM ではなく、`/start` と同じギルド内チャンネル**での実行が必須です。

### ブルートフォース対策

`/auth` は試行回数制限つきです。  
5 回連続で失敗すると、5 分間ロックされます。

## 認証後の保護

### 画面・URLの取り扱い

認証完了後、OBS ブラウザ上の認証コードは DOM から除去されます。  
また、可能な範囲で URL からトークンクエリを除去します（`public/script.js`）。

### 通信暗号化

認証後のメッセージは AES-256-GCM で暗号化されます。  
クライアント側は Web Crypto API で復号します。

### CORS

Socket.io の CORS ポリシーは `utils/corsPolicy.js` に集約されています。  
詳細は [ALLOWED_ORIGINS.md](./ALLOWED_ORIGINS.md) を参照してください。

## コマンド権限（公開コマンドの概要）

内部運用用コマンドは、この公開ドキュメントでは意図的に詳細を記載しません。

| コマンド | 権限の考え方 |
|---|---|
| `/status` / `/help` | **グローバルブラックリストに入っていないユーザー**が実行可能 |
| `/my-status` | グローバル BL でも照会可能（サーバー側で照会機能が有効な場合） |
| `/start` / `/auth` | **サーバーオーナー・管理者**、または **`/setup` で許可された**ロール／ユーザー |
| `/session` | **グローバル BL 対象外**ユーザーが実行可能。変更対象は **自分が作成したセッションのみ** |
| `/config` | **サーバーオーナー・管理者**のみ |
| `/setup` | サーバーオーナー・管理者、または `/config` で許可された担当 |
| `/blacklist` | サーバーオーナー・管理者、または `/config` の `ctrl_blacklist_*` で許可されたユーザー |
| `/global_blacklist` | `.env` の `BOT_OWNER_ID` と一致するユーザーのみ |

## ブラックリスト

- **グローバルBL**: コマンド実行とオーバーレイ表示の両方を遮断（`/my-status` の照会は例外）。
- **ローカルBL（サーバー）**: OBS への表示と `/start` 利用を遮断。

## データと保存

設定・セッション・許可リストは SQLite（既定: `app.db`）に保存されます。  
旧 `db.json` は条件付きで起動時マイグレーションされます（`database.js`）。

ギルド単位の一部設定は `MASTER_KEY` と guild id から派生した鍵で暗号化されます（`utils/deriveGuildKey.js`）。

Discord メッセージ本文は、暗号化して OBS に転送後、サーバー上に長期保存しません。
